# PRD：Timeline 密度、置信度优先与研究影响门禁

## 1. 当前问题

1. Event 总量继续增长时，Timeline 一次创建全部月份和卡片 DOM，初始样式计算与交互注册会逐步变慢。
2. 单月超过 6 条后仍全部铺开，读者难以识别该月真正重要的变化。
3. 事件顺序没有明确把置信度作为第一优先级，语言偏好可能干扰事实可靠性判断。
4. 现有当月论文组曾要求 2022 年 11 月以来每月补足至少 4 篇，导致编辑目标从“识别重大研究”异化为“填满月份”。内容长度和人工影响分也无法证明论文已经影响行业或某个研究领域。

## 2. 产品目标

- Event 总量超过 500 时，Timeline 只挂载最近 6 个月；更早月份以月份为组在接近视口时懒挂载，搜索或筛选时一次补齐待检索月份。
- 单月表层条目超过 6 个时，默认展示权重最高的 6 个条目，并提供月级展开 / 收起；筛选或搜索命中时自动显示全部匹配项。
- Timeline 以置信度为第一排序键，再比较影响、价值、热度与时间；中英文不设比例或加权。
- 同日论文组升级为当月论文组；一个月最多展示 10 篇通过研究影响门禁的研究，也可以是 0 篇。
- 取消任何逐月保底、连续月份覆盖率和语言配额。历史回填只为恢复真正重要的研究节点，不为填月降低质量。
- 当月论文组只在“全部变化”和“论文与研究”两个 Tab 出现；其他主线和“官方发布”不展示论文组。

## 3. 研究展示门禁

Timeline 论文组只消费同时满足“内容门禁”和“影响门禁”的公开 Event。

内容门禁：

- category 明确为 research / paper / research-paper，或证据中存在 arXiv 一手链接；
- 至少一个 Tier 1 / primary Evidence；
- confidence >= 80、impact >= 75、value >= 75；
- technical insight >= 80 字符、industry insight >= 50 字符、future outlook >= 40 字符；
- 不含占位内容。

影响门禁至少命中一条可审计路径。论文年龄决定可使用的证据，不能用成熟论文的绝对引用阈值淘汰当年新研究：

1. **成熟领域影响**：发表满一年后，OpenAlex 等开放学术索引显示达到绝对引用量阈值，并通过标题身份匹配。
2. **早期重要性**：发表不满一年、引用尚未形成时，必须同时具备直接官方研究发布、可读取的论文正文，以及正式出版、公开代码 / 数据、真实部署或独立复现中的至少一项。仅有 arXiv、实验室署名或模型自评分不充分。
3. **同行认可**：OpenReview 或正式出版方证明被高质量会议 / 期刊接收，同时存在独立复现、后续工作或基准采用；“已接收”本身不是充分条件。
4. **产业影响**：官方产品、标准、模型、数据集或生产系统明确采用该方法，或者论文官方代码 / 模型在 GitHub、Hugging Face 等一手平台形成可验证的持续采用。
5. **人工重大影响覆盖**：编辑可以基于至少两个高质量证据做覆盖，但必须记录理由、证据 URL、审核时间和失效条件，不能只写主观分数。

引用量只是影响证据之一，不能单独替代论文身份核验、领域相关性和产业判断。OpenAlex 不得成为当年论文的单点依赖；新论文可以由经过审计的直接证据路径进入 Timeline。身份或证据互相冲突、直接证据失效时，论文保持 Event / 候选状态但不进入 Timeline。

这些是 Timeline 注意力门禁，不改变 Event 的公开状态，也不删除永久页和 Evidence。低于门槛的论文不会进入 Timeline 论文组；空月不展示空论文组。

## 4. 排序与数量

- 非论文事件按 confidence、impact、value、heat 的字典序权重排序；同分时按最近进展倒序，不使用语言加权。
- 论文先通过研究影响门禁，再按影响路径强度、置信度、impact、value 与时间排序；一个月最多保留前 10 篇，不使用语言加权，也没有最低数量。
- 当月论文组独立于“单月 6 条”普通事件配额，组内论文默认折叠，不按普通 Event 卡直接铺开。

## 5. 非目标

- 不改变数据库 Schema、Event 发布状态或证据 DTO。
- 不把中文语言偏好变成事实质量替代品。
- 不把 Google Scholar 抓取、搜索排名、社交热度或模型主观评分作为自动发布依据。
- 不因为月份为空而告警；只有高质量信源失效、报告过期或已展示论文失去影响证据时才告警。
- 不在浏览器中执行来源 HTML 或模型输出。
